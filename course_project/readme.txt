В данном проекте представлена модель хранения данных популярного веб-сервиса auto.ru.

Описываемая модель данных представляет собой базу данных auto_ru, состоящую из 20 таблиц:

- users, profiles - таблицы, содержащие информацию о пользователях ресурса;
- messages - таблица, хранящая сообщения, которыми обмениваются пользователи;
- vehicles, vehicles_category, vehicle_brands, vehicle_models, vehicle_generations, vehicle_series, vehicle_modifications, vehicle_equipments, vehicle_characteristics, vehicle_characteristic_values,
vehicle_options, vehicle_option_values - таблицы, в которых содержится информация о типах транспортных
средств, их категориях (для коммерческого транспорта и мототехники), марках, моделях, поколениях, типах кузовов, модификациях, комплектациях, а также их характеристиках и опциях, характерных для транспортных средств конкретной комплектации;
- ads - таблица, в которой хранятся объявления о продаже транспортных средств;
- reviews - таблица, содержащая отзывы пользователей ресурса о транспортных средствах;
- media - таблица, хранящая загружаемые пользователями изображения;
- photo_albums, photos - таблицы, необходимые, чтобы формировать из таблицы media списки изображений, относящихся к тем или иным объявлениям или отзывам.

 Список файлов курсового проекта:
 
 1. auto_ru_db.sql - скрипт создания структуры базы данных курсового проекта.
 2. auto_ru_data.sql - скрипт наполнения базы данными.
 3. auto_ru_ER.erd - файл ER-диаграммы БД, созданный в программе DBeaver.
 4. auto_ru_ER.png - ER-диаграмма БД (изображение с расширением .png).
 5. auto_ru_selects.sql - скрипт характерных выборок.
 6. auto_ru_views.sql - скрипт с представлениями.
 7. auto_ru_procedures_functions_triggers.sql - хранимые процедуры / триггеры.
